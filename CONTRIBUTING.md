# Contributing to CmdTools

First of all, thank you for your interest in contributing to CmdTools.

We welcome bug reports, feature suggestions, documentation improvements, and code contributions.

---

## Before You Start

Please:

* Read the LICENSE file.
* Search existing Issues before creating a new one.
* Open an Issue before starting work on large changes.

---

## Reporting Bugs

When reporting a bug, please include:

* CmdTools version
* Operating system
* Steps to reproduce the issue
* Expected behavior
* Actual behavior
* Screenshots or logs (if available)

---

## Suggesting Features

Feature requests are always welcome.

Please explain:

* What problem your idea solves.
* Why the feature would be useful.
* Any additional information or examples.

---

## Pull Requests

Before submitting a Pull Request:

* Ensure your code works correctly.
* Follow the existing coding style.
* Keep changes focused and easy to review.
* Test your changes before submitting.

Submitting a Pull Request does **not** guarantee that it will be merged.

The project maintainer reserves the right to accept, reject, or request changes to any contribution.

---

## Code of Conduct

Please be respectful and professional when interacting with other contributors.

Harassment, abusive behavior, spam, or malicious contributions will not be accepted.
